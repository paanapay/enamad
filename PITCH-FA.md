# سرویس داده اینماد
### دسترسی به دیتابیس زنده کسب‌وکارهای دارای نماد — پنل وب، وب‌سرویس API و CRM

> **نسخه ارائه و بازاریابی** — برای معرفی سرویس به مشتری، تیم فروش یا شبکه‌های اجتماعی

---

## یک جمله‌ای

**دسترسی اشتراکی به دیتابیس زنده اینماد — از طریق پنل، API یا CRM — بدون نصب، بدون سرور، بدون نگهداری فنی.**

---

## مدل کسب‌وکار

ما **سرویس** می‌فروشیم، نه نرم‌افزار یا سورس‌کد.

| ❌ نمی‌فروشیم | ✅ می‌فروشیم |
|-------------|-------------|
| سورس‌کد پروژه | دسترسی ماهانه به پنل وب |
| فایل دیتابیس خام | وب‌سرویس API با کلید دسترسی |
| نصب روی سرور مشتری | CRM بازاریابی (پیامک / ایمیل) |
| مسئولیت نگهداری زیرساخت | ربات تلگرام / بله (افزودنی) |

**ارزش ما:** تیم فنی، به‌روزرسانی روزانه داده، پایداری سرویس و ابزار آماده — مشتری فقط استفاده می‌کند.

---

## مشکل مشتری

- جمع‌آوری دستی اطلاعات اینماد غیرممکن است (کپچا در هر صفحه).
- ساخت و نگهداری اسکرپر و دیتابیس هزینه و ریسک فنی دارد.
- تیم فروش به **لیست هدفمند** نیاز دارد، نه پروژه نرم‌افزاری.
- یکپارچه‌سازی داده با CRM و کمپین بدون ابزار، پرهزینه است.

## راه‌حل ما (به‌صورت سرویس)

1. **داده را ما نگه می‌داریم** — جمع‌آوری و به‌روزرسانی خودکار
2. **شما از پنل استفاده می‌کنید** — جستجو، فیلتر، جزئیات
3. **یا از API مصرف می‌کنید** — در نرم‌افزار خودتان
4. **یا CRM را فعال می‌کنید** — کمپین پیامک، ایمیل، ثبت تماس

---

## خدمات قابل ارائه

### ۱. پنل وب (بسته پایه)

- اکانت اختصاصی با نام کاربری/رمز
- جستجو بر اساس دامنه، کسب‌وکار، مالک
- فیلتر: استان، شهر، امتیاز، تاریخ انقضا، نوع تلفن
- جزئیات دامنه + مجوزها + استعلام زنده
- رابط فارسی، تقویم شمسی

### ۲. وب‌سرویس API (بسته سازمانی)

- REST API با کلید دسترسی (API Key)
- جستجو، فیلتر، جزئیات دامنه
- محدودیت درخواست بر اساس بسته (مثلاً ۱۰۰۰/روز)
- مستندات فنی برای تیم توسعه مشتری
- مناسب: SaaS، اپلیکیشن، یکپارچه‌سازی با CRM داخلی

### ۳. CRM بازاریابی (بسته حرفه‌ای)

- کمپین پیامکی با کاوه‌نگار (قالب Lookup)
- کمپین ایمیلی SMTP
- قالب‌های شخصی‌سازی‌شده (نام کسب‌وکار، دامنه، تاریخ انقضا)
- انتخاب مخاطب از فیلتر پنل
- ثبت تماس تلفنی و گزارش ارسال

### ۴. ربات تلگرام / بله (افزودنی)

- جستجو و مرور سریع از داخل پیام‌رسان
- مناسب تیم‌های میدانی و فروش
- دسترسی محدود به کاربران مجاز

---

## برای چه کسانی؟

| مخاطب | سرویس پیشنهادی |
|--------|----------------|
| **آژانس دیجیتال مارکتینگ** | پنل + CRM (پیامک/ایمیل) |
| **تیم فروش B2B** | پنل + ربات |
| **شرکت SaaS / پرداخت** | API + پنل |
| **تحلیل‌گر بازار** | پنل (فیلتر و خروجی) |
| **استارتاپ** | API — تمرکز روی محصول خودشان |

---

## بسته‌های اشتراک

| بسته | شامل | مناسب |
|------|------|--------|
| **پایه** | پنل وب — جستجو، فیلتر، جزئیات، ۱ کاربر | تحقیق، تیم کوچک |
| **حرفه‌ای** | پایه + CRM + ربات، تا ۵ کاربر | آژانس، فروش |
| **سازمانی** | حرفه‌ای + API اختصاصی + SLA + پشتیبانی | شرکت‌های بزرگ |

> قیمت بر اساس تعداد کاربر، حجم API و نیاز CRM. **دمو ۷ روزه رایگان.**

## مزایای رقابتی (چرا سرویس ما؟)

| مزیت | توضیح |
|------|--------|
| **بدون نصب** | لاگین کنید و شروع کنید |
| **بدون نگهداری** | به‌روزرسانی و زیرساخت با ماست |
| **داده زنده** | هر روز تازه — نه اکسل قدیمی |
| **هزینه قابل پیش‌بینی** | اشتراک ماهانه، نه پروژه نرم‌افزاری |
| **بومی** | فارسی، شمسی، کاوه‌نگار، بله |
| **چند کانال دسترسی** | پنل، API، ربات، CRM |

## سناریوهای فروش

### سناریو ۱ — آژانس مارکتینگ
> «می‌خواهیم به فروشگاه‌های تهران با نماد در حال انقضا پیامک بفرستیم.»

✅ اشتراک حرفه‌ای → فیلتر در پنل → کمپین CRM → گزارش

### سناریو ۲ — استارتاپ SaaS
> «می‌خواهیم در اپلیکیشنمان وضعیت اینماد یک دامنه را نشان دهیم.»

✅ اشتراک سازمانی → API → یکپارچه‌سازی در محصول

### سناریو ۳ — تیم فروش
> «هر روز لیست فروشگاه‌های جدید را ببینیم و تماس بگیریم.»

✅ اشتراک پایه یا حرفه‌ای → پنل + ثبت تماس CRM
